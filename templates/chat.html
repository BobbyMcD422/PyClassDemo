GET and POST

Class chat

This page shows how form data moves from the browser to Flask, gets saved in a SQLAlchemy model, and comes back as generated HTML.

POST /chat

The browser sends this message to Flask. SQLAlchemy saves it as a Message model.

GET /chat

  1. SQLAlchemy loads recent Message models.
  2. Jinja places those messages into chat.html.
  3. The browser receives a new HTML page.
  4. Refreshing the page runs the GET request again.
